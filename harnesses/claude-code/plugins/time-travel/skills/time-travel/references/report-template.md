# Report template

The canonical skeleton for the user-facing report. Generated in Phase 5 as `<YYYY-MM-DD-HHMM>-<slug>-report.md`.

## How to use this template

1. Fill in every section. Don't omit sections — empty placeholders signal a process miss.
2. Substitute `{braces}` with real values.
3. Keep code-block markdown fences intact.
4. If a section truly has no content (e.g., no Buried risks found — though this is suspicious; re-examine), write `_No items in this category — see process log for synthesis notes._` rather than deleting the section.
5. The TL;DR is the only section most readers will read. Make it count.

## Length expectations

- TL;DR: 3 bullets + confidence line + Buried-risk line. Under 80 words total.
- User-POV pass: 4 bullets, each 1–2 sentences. Under 200 words total.
- Timeline narratives: 100–200 words per cell. Total ≈ 1500–2400 words for 5–7 personas × 3 horizons.
- Risk classification tables: as needed.
- Revised plan: same length ballpark as the original plan, plus ~20% for injections.
- Evidence pack: under 600 words.

Total report length typically **2500–4000 words**. If much shorter, content is missing. If much longer, you're padding — cut.
